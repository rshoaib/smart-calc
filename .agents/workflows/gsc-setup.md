# Google Search Console Setup

This workflow automates the "Day 1" Google Search Console setup for a new website. It uses a secure browser subagent to add the URL Prefix property and immediately submit the `sitemap.xml`.

## How to Use

Say: `@[/gsc-setup] https://your-new-site.com`

---

## 🤖 Automated Subagent Instructions

When the user triggers this workflow, start a `browser_subagent` with the following `Task`:

```text
Navigate to Google Search Console: https://search.google.com/search-console/welcome
1. Under the "URL prefix" box on the right, enter the full domain requested by the user (e.g., https://buildwithriz.com/).
2. Click "CONTINUE".
3. Wait for the "Ownership auto-verified" dialog (assuming DNS/Domain is already under the same Google account or site is deployed on Vercel/Next.js with matching GA4).
4. Click "GO TO PROPERTY".
5. In the left sidebar of the new property dashboard, click on "Sitemaps".
6. In the "Add a new sitemap" input field, type `sitemap.xml`.
7. Click "SUBMIT".
8. Wait for the "Sitemap submitted successfully" popup and click "GOT IT".
9. Verify the sitemap says "Success" in the submitted list.
10. Return when completed.
```

**TaskName**: `Setup GSC Property`
**TaskSummary**: `Add property and submit sitemap for the requested domain in Google Search Console.`
**RecordingName**: `gsc_initial_setup`

---

## ✅ Final Deliverable

After the subagent successfully returns:
1. Embed the `.webp` recording generated by the subagent into the chat so the user can verify the property was added.
2. Confirm the exact URL that was submitted.
3. Advise the user to wait 24-48 hours for data to begin populating before running the `@[/gsc-audit]` workflow.
